# 전세사기 예방 앱 구축 가능성 분석 및 기획서

현재 보유하고 있는 대한민국 법률 및 행정규칙 데이터베이스(`legalize-kr`, `admrule-kr`)를 기반으로 전세사기 예방 앱을 만들 수 있는지 분석하고, 필요한 구성 요소들을 정리한 기획서입니다.

---

## 1. 현재 코드 및 데이터로 가능한 범위

보유 중인 데이터는 대한민국의 **모든 법률 및 행정규칙의 텍스트 데이터베이스**입니다. 이를 활용하면 다음과 같은 기능을 구현할 수 있습니다.

### 💡 RAG(검색 증강 생성) 기반 법률 상담 챗봇
* **설명**: 사용자가 "전세 계약 후 확정일자를 받았는데 임대인이 당일에 저당권을 설정하면 어떻게 되나요?"와 같은 질문을 했을 때, `주택임대차보호법` 및 관련 판례/행정규칙 데이터를 실시간으로 참조하여 정확한 법률적 조언을 제공하는 AI 챗봇을 만들 수 있습니다.
* **핵심 이점**: 일반적인 LLM이 겪는 환각 현상(Hallucination) 없이, 실제 법령에 근거한 신뢰성 높은 답변을 생성할 수 있습니다.

### 📝 전세 계약 맞춤형 특약 사항 추천
* **설명**: 임차인에게 불리한 조항을 걸러내고 법적으로 자신을 보호할 수 있는 필수 특약 사항(예: 국세/지방세 완납 증명 제출 의무, 대항력 확보 전 담보권 설정 금지 등)을 법령에 기반하여 자동으로 추천해 줍니다.

### 📘 단계별 법률 가이드 및 체크리스트
* **설명**: 전세 계약 전, 계약 당일, 계약 후 각 단계별로 확인해야 하는 법적 의무 사항과 대항력 발생 요건 등을 시각적으로 정리한 체크리스트를 제공합니다.

---

## 2. 실질적인 전세사기 방지를 위해 '추가로' 필요한 것들

단순히 법률 조문을 아는 것만으로는 실제 교묘하게 발생하는 전세사기를 완벽히 막을 수 없습니다. 전세사기는 대부분 **매물의 실시간 권리관계 유출, 시세 조작, 임대인의 신용 문제**에서 발생하기 때문에, 이를 방지하려면 외부 데이터 연동이 필수적입니다.

### 🔍 1) 외부 데이터 및 API 연동
| 필요 데이터 | 제공 기관 / API 예시 | 앱에서의 활용 목적 |
| :--- | :--- | :--- |
| **부동산 등기부등본** | 대법원 인터넷등기소 (민간 스크래핑/조회 API 활용) | 근저당권 설정 여부, 선순위 채권 금액 확인, 압류/가압류 등 권리관계 실시간 분석 |
| **건축물대장** | 정부24 (공공데이터포털 API) | 위반건축물 여부 확인 (위반건축물은 전세대출 및 보증보험 가입이 불가능함) |
| **주택 실거래가 및 시세** | 국토교통부 실거래가 API, KB부동산 | 해당 매물의 전세가율(전세가/매매가)을 계산하여 '깡통전세' 위험도 측정 |
| **임대인 세금 체납 정보** | 국세청 / 행정안전부 API | 임대인의 미납 국세/지방세 여부 확인 (임대인 체납 세금은 보증금보다 우선 변제되므로 필수 확인 필요) |
| **악성 임대인 명단** | HUG(주택도시보증공사) 안심전세 API | 과거 보증금을 돌려주지 않은 악성 임대인(블랙리스트) 여부 조회 |

### 🛠️ 2) 추가 개발이 필요한 기술 스택
1. **백엔드 서버 (API 중계 및 분석 엔진)**
   * 등기부등본 및 정부 공공 API에서 데이터를 받아와 위험도 점수(Risk Score)를 계산하는 백엔드 로직이 필요합니다. (Node.js, FastAPI 등)
2. **벡터 데이터베이스 (Vector DB)**
   * 법률 마크다운 문서들(`legalize-kr`, `admrule-kr`)을 임베딩하여 빠르고 정확하게 의미론적 검색을 수행할 수 있도록 지원하는 DB(예: Chroma, Pinecone, pgvector 등)가 필요합니다.
3. **사용자 인터페이스 (Frontend)**
   * 모바일 앱(Flutter, React Native 등) 또는 반응형 웹앱(Next.js, Tailwind CSS 등)을 통해 사용자가 주소만 입력하면 직관적으로 위험도를 파악할 수 있는 화면이 필요합니다.

## 3. 추천 앱 아키텍처 및 핵심 기능안

### 1단계: 주소 입력 및 매물 진단 (가장 핵심)
1. 사용자가 매물 주소와 전세 보증금 액수를 입력합니다.
2. 앱이 API를 통해 **등기부등본**과 **건축물대장**을 조회합니다.
3. 등기부등본 상의 채권최고액(융자) + 선순위 보증금 + 사용자의 보증금 합산액이 주변 시세의 몇 %인지 분석합니다. (80% 이상일 경우 깡통전세 경고)
4. 미납 국세 여부 및 위반건축물 여부를 종합하여 **"안전성 등급(정상/주의/위험)"**을 도출합니다.

### 2단계: 법률 챗봇 및 특약 생성 (보유 데이터 활용)
1. 진단 결과에 따라 발생할 수 있는 법적 위험(예: 선순위 근저당으로 인한 경매 시 보증금 변제 순위)을 설명합니다.
2. 계약서 작성 시 임대인에게 요구해야 할 **특약 사항 마크다운 문서**를 자동 생성하여 복사할 수 있도록 제공합니다.
3. 사용자가 전세 계약과 관련하여 궁금한 점을 질문하면 법령 기반으로 답변하는 챗봇 서비스를 제공합니다.

## 4. 향후 로드맵 제안

1. **Phase 1: 지식 베이스 구축 (현재 데이터 활용)**
   * 현재 보유한 대한민국 법률 데이터를 Vector DB에 적재하고, LLM과 연동하여 **전세사기 전문 법률 상담 챗봇**을 먼저 구현합니다.
2. **Phase 2: 공공데이터 연동**
   * 국토교통부 실거래가 및 건축물대장 API를 연동하여 시세 대비 전세가율을 계산하는 **깡통전세 계산기**를 구축합니다.
3. **Phase 3: 권리 분석 자동화**
   * 등기부등본 PDF/텍스트 파싱 API를 도입하여 등기부등본 상의 위험 요소를 자동으로 추출하는 **권리 분석 엔진**을 적용합니다.
