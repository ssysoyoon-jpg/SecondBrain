# 전세가드 AI 웹 앱 배포 가이드라인 (Deployment Guide)

제작 완료된 Next.js 기반 전세가드 AI 프로토타입 웹 애플리케이션을 실제 인터넷에 배포(무료)하여 스마트폰이나 다른 컴퓨터에서 접속할 수 있도록 만드는 가이드라인입니다.

Next.js 어플리케이션 배포에 가장 최적화된 **Vercel (버셀)** 플랫폼을 이용하는 두 가지 방법을 제시합니다.

---

## 🛠️ 방법 A: Vercel CLI를 이용한 초간편 배포 (로컬 터미널 활용)

깃허브(GitHub) 가입 없이 로컬 터미널 창에서 명령어 몇 줄만으로 즉시 배포할 수 있는 방법입니다.

### 1단계: Vercel 도구 설치
로컬 터미널 창(c:\SecondBrain\jeonse-guard 위치)을 열고 아래 명령어를 입력하여 Vercel CLI(명령줄 도구)를 전역에 설치합니다.
```bash
npm install -g vercel
```

### 2단계: Vercel 로그인 및 인증
아래 명령어를 입력합니다.
```bash
vercel login
```
* 화면에 로그인 방식 선택창이 뜹니다. **`Continue with Email`** 또는 본인의 편리한 계정(Google 등)을 선택해 연동 로그인합니다.
* 브라우저창이 열리며 인증이 완료되면 터미널에 성공 메시지가 나타납니다.

### 3단계: 배포 명령 실행
프로젝트 폴더 내에서 아래 명령을 실행합니다.
```bash
vercel
```
* **터미널 질문 세부 설정 요령**:
  1. *Set up and deploy “c:\SecondBrain\jeonse-guard”?* -> **`Y`** (엔터)
  2. *Which scope do you want to deploy to?* -> 본인의 스코프 선택 (기본값 엔터)
  3. *Link to existing project?* -> **`N`** (새로운 프로젝트이므로 엔터)
  4. *What’s your project’s name?* -> **`jeonse-guard`** (기본값 엔터)
  5. *In which directory is your code located?* -> **`./`** (기본값 엔터)
  6. *Want to modify these settings? (Build Command 등)* -> **`N`** (기본값 엔터)
* 빌드가 진행된 후, 단 1분 만에 실시간 웹 페이지 접속용 임시 주소(Preview URL)가 발급됩니다.

### 4단계: 실 운영 배포 (Production Deploy)
테스트가 끝난 뒤, 정식으로 최종 도메인을 확보하려면 아래 명령을 쳐서 최종 배포를 마무리합니다.
```bash
vercel --prod
```
* 최종 완료되면 영구적으로 사용 가능한 실시간 배포 URL(예: `https://jeonse-guard.vercel.app`)이 제공됩니다.

## 🐙 방법 B: GitHub 연동을 이용한 지속적 배포 (강력 권장)

깃허브 저장소에 코드를 올린 뒤 Vercel 웹사이트와 연동하는 방법입니다. 코드가 수정될 때마다 깃허브에 올리기만 하면 **인터넷상의 페이지가 실시간 자동 업데이트**되므로 유지보수에 매우 유리합니다.

### 1단계: GitHub에 코드 올리기
1. 깃허브([github.com](https://github.com/))에 로그인한 뒤, 우측 상단 `+` 버튼을 눌러 **[New repository]**를 만듭니다. 이름은 `jeonse-guard`로 설정합니다.
2. 로컬 터미널 창을 열고 아래 명령을 한 줄씩 실행하여 깃허브에 코드를 푸시합니다:
   ```bash
   git init
   git add .
   git commit -m "feat: 전세가드 AI 올인원 대시보드 배포"
   git branch -M main
   git remote add origin https://github.com/본인계정명/jeonse-guard.git
   git push -u origin main
   ```

### 2단계: Vercel 웹사이트 연동
1. **[Vercel 공식 사이트](https://vercel.com/)**에 접속하여 회원가입 후 로그인합니다. (GitHub 계정으로 가입하시는 것이 좋습니다.)
2. 대시보드 화면 우측 상단의 **[Add New...]** -> **[Project]**를 클릭합니다.
3. 깃허브 연동을 통해 방금 만든 `jeonse-guard` 저장소(Repository) 옆의 **[Import]** 버튼을 누릅니다.

### 3단계: 환경 설정 및 배포 완료
1. `Configure Project` 설정 화면이 나옵니다. 다른 설정은 건드릴 필요가 없습니다.
2. 만약 OpenAI API Key를 웹 화면 설정 창에서 매번 입력하는 대신, 서버 측에 안전하게 아예 내장하고 싶으시다면 아래 **`Environment Variables`** 칸에 다음과 같이 환경 변수를 채워 넣습니다:
   * **Name**: `OPENAI_API_KEY`
   * **Value**: `본인의 sk-... 로 시작하는 API Key`
3. 하단의 **[Deploy]** 단추를 누릅니다.
4. 약 1분간 빌드가 완료되면 폭죽 애니메이션과 함께 인터넷을 통해 전 세계 누구나 접속할 수 있는 **무료 영구 도메인 주소**가 발급됩니다.
